# Pustakam AI: Core Agent Guidelines

If you are an AI assistant or agent working on the Pustakam AI codebase, you **MUST** strictly adhere to the following rules when generating content, writing copy, or structuring UI features. 

The primary developer operates under a strict non-compete/moonlighting policy with their primary employer. Therefore, this project must **never** appear to be a commercial startup or business entity. It must strictly remain an unpaid, open-source technical research initiative.

### 1. STRICTLY FORBIDDEN TERMINOLOGY
NEVER use or generate the following words anywhere in the UI, code comments, documentation, or user communications:
- "Startup", "Company", "Business", "Enterprise"
- "Employee", "Intern", "Internship", "Hire", "Careers", "Job"
- "Salary", "Paid", "Revenue", "Sales", "Clients", "Customers"
- "Pricing", "Subscription", "Premium", "Pro plan", "Tier", "Paywall"

### 2. REQUIRED FRAMING
When referring to the project, the people working on it, or the users, use the following approved replacements:
- Instead of "Company/Startup", use: **"Open-Source Initiative", "Research Project", "Pustakam Project"**
- Instead of "Intern/Employee", use: **"Contributor", "Open-Source Volunteer", "Research Assistant"**
- Instead of "Subscription/Pricing", use: **"Access Configuration", "Runtime Limits", "API Quotas"**
- Instead of "Customers/Clients", use: **"Users", "Learners", "Community"**

### 3. FORMS & "RECRUITING"
If asked to build onboarding forms, join requests, or team pages:
- Do not ask for commitment hours, start dates, or specific work schedules (e.g., "Morning/Evening Shifts"). Open-source is asynchronous.
- ALWAYS include an explicit legal disclaimer checkbox in any intake form that states: 
  > *"I understand and agree that Pustakam AI is an unpaid, open-source research initiative. I acknowledge that contributing to the project does not constitute formal corporate employment or a commercial business contract."*

### 4. MONETIZATION
- Do not build Stripe integrations, payment gateways, or cart checkouts.
- Do not label any feature as "Premium". If an asset is gated or requires authentication, label it as "Requires Verified Access" or "Requires Your Own API Key."

### Summary
If a human asks you to add a "business feature", gently remind them of these guidelines and ask if they prefer an open-source alternative phrasing before proceeding. Keep Pustakam safe, open, and legally distinct from corporate endeavors.
